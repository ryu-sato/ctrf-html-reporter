# ReportInfo Component Example

## Basic Usage

```vue
<script setup>
import { data as report } from './report.data.js';
</script>

<ReportInfo :report="report" />
```

## Empty State

```vue
<ReportInfo :report="null" />
```

## Custom Report

```vue
<script setup>
const customReport = {
  reportFormat: "CTRF",
  specVersion: "1.0.0",
  reportId: "custom-id-123",
  timestamp: new Date().toISOString(),
  generatedBy: "Custom Reporter v1.0.0",
  results: {
    tool: {
      name: "custom-tool",
      version: "1.0.0"
    },
    summary: {
      tests: 10,
      passed: 8,
      failed: 2,
      skipped: 0,
      duration: 5000
    }
  },
  extra: {
    customField: "Custom Value",
    anotherField: 123
  }
};
</script>

<ReportInfo :report="customReport" />
```

## Features

### Automatic Duration Calculation

The component automatically handles duration in multiple ways:
1. Uses `summary.duration` if available
2. Calculates from `summary.start` and `summary.stop` if duration is not provided
3. Shows "N/A" if neither is available

### Optional Field Handling

All fields are optional. The component will only display fields that exist in the report object:
- Report ID
- Generated By
- Timestamp
- Tool (name and version)
- Duration
- Report Format
- Spec Version
- Extra fields (any custom properties)

### Responsive Layout

The component uses a responsive grid layout that adapts to different screen sizes:
- Desktop: Multiple columns
- Mobile: Single column

### Custom Fields via Extra

Any fields in the `report.extra` object will be automatically displayed with proper formatting:
- CamelCase → Title Case
- snake_case → Title Case
- Object values are JSON stringified
